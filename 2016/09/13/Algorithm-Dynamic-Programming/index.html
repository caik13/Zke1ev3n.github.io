Well, let’s turn our attention to the theme of this article.The following is the description of the wiki on dynamic programming.

Dynamic programming (also known as dynamic optimization) is a method for solving a complex problem by breaking it down into a collection of simpler subproblems, solving each of those subproblems just once, and storing their solutions - ideally, using a memory-based data structure. The next time the same subproblem occurs, instead of recomputing its solution, one simply looks up the previously computed solution, thereby saving computation time at the expense of a (hopefully) modest expenditure in storage space. (Each of the subproblem solutions is indexed in some way, typically based on the values of its input parameters, so as to facilitate its lookup.) The technique of storing solutions to subproblems instead of recomputing them is called “memoization”.

Dynamic Programming is typically used to find the optimal solution.It is more like a thought instead of a specific algorithm, unlike other search or numerical algorithm, dynamic programming does’t have a standard mathmatical expression or a clear problem-solving method.Different problems have various models and specific analysizing.

Let’s get start with a simple and classic example:

Given coins with values 1, 3, and 5.And the sum S is set to be 11. Find the minimum number of coins the sum of which is S.

Review the description of wiki above, the main idea is decomposing the problem into subproblems.So before we discuss the sum 11, we first discuss the sum i.If we set the i to 0, it’s clear that the minimum number of coins is 0, and in order to express more clearly, we use d(i) = j to indicate that the sum i needs at least j numbers of conins.And we already got d(0) = 0.We then go to sum 1, and we see that only coin 1 is less than or equal to the current sum, so we firstly pick up the coin 1, the remianing sum is 0, it means we should find what’s the minimum number of coins of the sum 0.However, we have found it just now, so we’ll have a solution with 1 coin for sum 1.It’s the only solution yet found for this sum. We mark it as d(1) = d(1-1) + 1 = d(0) + 1 = 0 + 1 = 1.Then we proceed to the next step, the sum 2. We again see that the only coin which is less or equal to this sum is the first coin, having a value of 1. We pick up the coin 1, the remaining sum is 1, we also had the answer of sum 1. So d(2) = d(2 - 1) + 1 = d(1) + 1 = 1 + 1 = 2.Now we proceed to sum 3.We now have 2 coins which are to be analyzed – first and second one, having values of 1 and 3. If we pick up the coin 1, remaining sum is 3 -1 = 2, so the solution is d(3) = d(3 - 1) + 1 = d(2) + 1 = 3，it means we can pick up 3 coins with value 1. Now let’s take the second coin with value equal to 3. The sum for which this coin needs to be added to make 3 , is 0. We know that sum 0 is made up of 0 coins. Thus we can make a sum of 3 with only one coin – 3. This solution is d(3) = d(3 - 3) + 1 = d(0) + 1 = 1.We see that it’s better than the previous found solution for sum 3 , which was composed of 3 coins. We can use an expression to express this: d(3)=min{d(3-1)+1, d(3-3)+1}.Then we update the solution and mark it as having only 1 coin.The same we do for sum 4, and get a solution of 2 coins – 1+3. And so on.

Now we should learn the two most important concepts of dynamic programming: State(状态) and Bellman Equation(状态转移方程).The above d(i) means the minimum number of of coins of sum i, and it’s the state of this problem.It’s a way to describe a situation, a sub-solution for the problem.So how could we find the state? It’s simple, for each coin j, Vj<=i, look at the minimum number of coins found for the i-Vj sum (we have already found it previously). Let this number be m. If m+1 is less than the minimum number of coins already found for current sum i, then we write the new result for it.And the Bellman Equation for the problem is d(i) = min{d(i - Vj) + 1}, for example, d(3) = min{d(3 - 1) + 1, d(3 - 3) + 1}.

We already have the State and the Bellman Equation, then we can write the code to sovle this problem.Dynamic Programming code is always very short, usually just several lines, but not easy to understand.

As a result we have found a solution of 3 coins which sum up to 11.
